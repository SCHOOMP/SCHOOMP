# Graham Schomp 🎮

## 💫 About Me
I'm a software engineer pivoting into **data engineering** — I build data pipelines that move messy, real-world data into clean, queryable models. Day to day I work with Python, SQL, and the modern data stack (dbt, Airflow, Docker), and I have hands-on experience designing ETL/ELT processes and optimizing SQL against large datasets in production.

I'm currently building a containerized, end-to-end data pipeline as a portfolio project (see below), and I came up through full-stack development, so I'm comfortable across the whole system — from the database to the API to the dashboard. Outside of work I'm a lifelong gamer and tinkerer, which is where the curiosity that drives all of this started.

🔧 **Currently building:** [food-data-pipeline](https://github.com/SCHOOMP/food-data-pipeline) — an ETL pipeline pulling from the Open Food Facts API (Python → dbt → Airflow, containerized with Docker).
